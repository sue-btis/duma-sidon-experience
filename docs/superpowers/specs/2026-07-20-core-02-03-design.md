# CORE-02 and CORE-03 home experience

## Scope

Adapt the existing progressive map. Do not replace its SVG asset or its staged reveal model, and do not read either wireframe PNG. The written CORE-02 and CORE-03 specifications are canonical.

The home sequence becomes CORE-01, CORE-02, CORE-03, then the existing industry, client, and partner material as CORE-04 evidence.

## Scene model

CORE-02 is a pinned, scroll-controlled scene. Its timeline is primary: five capability chapters appear in order and remain visible as accumulated context. The existing map stays behind it, beginning at Chihuahua and growing quietly in parallel. It includes the approved bilingual headline, supporting copy, and 35+ proof point. It names radios and antennas at origin but does not reveal Integración, Sidón, Duma, product categories, client logos, module structure, or location counts.

CORE-03 follows without a visual reset. It reuses the same map markup and progressive activation model, brings it into the foreground, and leaves a recognizable but subdued CORE-02 timeline behind it. Geographic stages are Chihuahua, Mexican locations, the cross-border group of Arizona, California, and Texas, then Central America and the complete footprint. Chihuahua remains the only persistent map label. Other approved locations use unlabeled pins. No pin asserts an office, and no counters, flags, customer logos, or coverage-style sales graphics appear.

At CORE-03 completion, the map briefly stabilizes, then the orbit directs a zoom toward Chihuahua. The existing industry, client, and partner scene follows as CORE-04 operational proof.

## Implementation boundaries

Keep the map decompression and SVG bucket animation in `MapExperience`. Extend its API with CORE-02 and CORE-03 copy and stage metadata rather than adding a map library or a second map asset. Add a focused timeline presentation component only if it keeps the map component legible. Keep translations in the existing `home.json` modules for English and Spanish.

Use native scroll progress, CSS transforms and opacity, and the existing request-animation-frame smoothing. Stage thresholds must clamp progress and preserve sequence on fast forward and backward scroll. For reduced motion, every state remains readable while transitions become immediate.
