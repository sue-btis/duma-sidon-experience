# CORE-04 operational credibility

## Scope

Replace the current home industries scene with CORE-04. It remains a single static-export home section between the map (CORE-03) and worlds (CORE-05); it adds no routes, data sources, or third-party dependencies.

## Content hierarchy

1. **Industries** leads: seven concise icon-and-label operating environments: Retail; Restaurants & Food Service; Industrial & Manufacturing; Banking & Financial Services; Corporate & Commercial Spaces; Logistics & Distribution; and Hospitality.
2. **Selected clients** follows as one curated, unsegmented field. Use the supplied provisional working set only where an existing local approved asset is present. No client is permanently assigned to an industry.
3. **Technology partners** is a separate technical-enablement layer. It uses the supplied provisional partner set and is never mixed with clients.

English uses the supplied CORE-04 headline and supporting copy. Spanish receives an equivalent translation in the existing locale message files.

## Interaction and layout

The existing home scene observer remains the trigger for staged CSS reveal. Industries reveal first, client proof next, then partners. The active layer is most prominent; earlier layers reduce slightly in prominence. No new scroll library or client-side state is needed.

Clients and partners render as independent, continuous horizontal logo bands. They move in opposite directions. Hovering a band, or keyboard-focusing one of its logos, pauses that band. Logos retain their source proportions and accessible alt text. On smaller screens, the section becomes sequential with horizontally clipped bands; it does not introduce a logo carousel control.

The map, timeline, and existing orbit visuals remain background context. This change does not create literal industry scenes, permanent client-to-industry mapping, new orbit topology, or a CORE-05 transition system because those capabilities are not present in the current implementation.

## Validation

Run `pnpm lint` and `pnpm build` from the repository root. Manually check `/en/` and `/es/`: the three layers are ordered correctly, the client and partner bands remain separate, hover/focus pauses the relevant band, and logos are not distorted.
